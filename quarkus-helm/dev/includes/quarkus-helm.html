Configuration property fixed at build time - All other configuration properties are overridable at runtime

Configuration property

Type

Default

quarkus.helm.enabled

Enabled the generation of Helm files.

boolean

true

quarkus.helm.name

Name of the Helm chart. If not set, it will use the application name.

string

quarkus.helm.version

Version of the Helm chart. If not set, it will use the application version.

string

quarkus.helm.description

The Helm chart single-sentence description.

string

quarkus.helm.icon

Icon of the Helm chart. It must be a URL to an SVG or PNG image.

string

quarkus.helm.home

Project’s home page of the Helm chart. It must be a URL.

string

quarkus.helm.keywords

List of keywords to add to the chart.

list of string

quarkus.helm.sources

The Helm chart list of URLs to source code for this project.

list of string

quarkus.helm.extension

Extension of the Helm tarball file. Default is tar.gz.

string

tar.gz

quarkus.helm.create-tar-file

If Helm tar file is generated.

boolean

false

quarkus.helm.maintainers."maintainers".name

Name of the maintainer.

string

required

quarkus.helm.maintainers."maintainers".email

Email of the maintainer.

string

quarkus.helm.maintainers."maintainers".url

URL profile of the maintainer.

string

quarkus.helm.dependencies."dependencies".name

Name of the dependency.

string

required

quarkus.helm.dependencies."dependencies".version

Version of the dependency.

string

required

quarkus.helm.dependencies."dependencies".repository

Repository of the dependency.

string

required

quarkus.helm.dependencies."dependencies".alias

Alias of the dependency.

string

quarkus.helm.values."values".property

The name of the property that will be present in the Helm values file.

string

required

quarkus.helm.values."values".json-paths

A comma-separated list of [JSONPath](https://tools.ietf.org/id/draft-goessner-dispatch-jsonpath-00.html) expressions to map the Dekorate auto-generated properties to the final Helm values file.

list of string

required

quarkus.helm.values."values".profile

The profile where this value reference will be mapped to. For example, if the profile is dev, then a values-dev.yml file will be created with the value.

string

quarkus.helm.values."values".value

The value that the property will have in the Helm values file. If not set, the extension will resolve it from the generated artifacts.

string
